Multi-User VR Laboratory

VR Development

Unity, Blender, Photon Fusion, Oculus Quest, Rokoko SmartSuit Pro

February 2024 - May 2025

This project is a multi-user virtual reality laboratory experience developed as my senior capstone project. The goal was to design a shared VR environment where multiple users can join the same session, interact with lab equipment in real time, and collaboratively perform experimental procedures. The project was built in Unity for Oculus Quest headsets, and Meta provided the hardware support specifically for this project. Real-time networking and synchronization were implemented using Photon Fusion.

Background & Motivation

Most VR educational tools focus on single-user interactions. While these systems are useful for learning basic concepts, they fail to simulate the collaborative nature of real laboratory environments, where communication and shared activity are essential.

We aimed to bridge this gap by creating a VR environment that supports true multi-user interaction, synchronized object states, and collaborative workflows. The experience focuses on laboratory tasks that require coordination, such as transferring liquids between test tubes, preparing chemical solutions, and using scientific tools.

Development Process

The environment and interactions were developed in Unity, with a physics-based system that allows users to grab objects, manipulate lab tools, and observe changes in real time. All major interactions—holding objects, changing liquid levels, activating equipment—are network-synchronized so each user sees the same state.

I implemented core interaction logic, collider-based fluid manipulation, object ownership transfer through Photon, and custom VR controller handling.

Networking & Multi-User Experience

We used Photon Fusion to build stable real-time connections between users. Each participant appears in the environment as an avatar and can observe others' actions. Object states, transforms, and procedural steps are replicated across the network, enabling shared workflows.

This multi-user design allows learners to communicate, follow each other's actions, and complete tasks together, reflecting real-life lab settings more accurately than single-user VR experiences.

Motion Capture & Realistic Animation

To enhance realism, we recorded the movements of an actual laboratory worker using a Rokoko motion capture suit. The captured data was processed and cleaned in Blender, then retargeted to a humanoid rig used in Unity.

We also modeled the avatar's face to resemble the real lab worker (with permission), allowing the VR demonstrator to reflect an authentic human presence. This figure performs proper lab procedures inside the environment, guiding users and demonstrating workflows they later replicate.

This grounded motion data made the experience feel more natural and professional, especially for instructional sequences.

Hardware & Tools

The project was developed using:

  • Unity (C#)
  • Photon Fusion
  • Oculus Quest
  • Blender
  • Rokoko SmartSuit Pro

We tested interactions directly on headset hardware throughout development.

Outcome

The final system demonstrates how VR can support collaborative scientific training by combining realistic motion, networked interactions, and immersive spatial design. The environment allows users to work together, observe demonstrations, and learn hands-on procedures in a virtual space.

The project serves as a foundation for scalable VR laboratory education, with potential future directions including AI-assisted instruction, expanded procedural content, and extended multi-role lab simulations.
